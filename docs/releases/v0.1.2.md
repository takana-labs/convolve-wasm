# @takana-labs/convolve-wasm v0.1.2

**Status:** Draft candidate. Tagging and JSR publication are blocked until the immutable v0.1.1 Android record and this v0.1.2 record each explicitly say **Pass**.

## Lower-memory full FFT

- Retains the v0.1.1 post-decode, pre-worker safe-rejection plan and device-class browser budgets.
- Removes the materialized reverse waveform and full beat-pan vector: panning is incremental, reverse is a virtual palindrome, and normalization traverses the layered result.
- Stores each convolution channel at its exact linear length and reuses explicit FFT scratch storage.
- Streams the byte-compatible 68-byte WAVE_FORMAT_EXTENSIBLE PCM24 WAV in ordered 65,536-frame chunks through a pull-based worker session. The client builds the final `audio/wav` Blob backpressured rather than accepting a redundant whole-WAV copy.
- Preserves the public API, errors, progress stages, metadata, full-engine WAV bytes, stereo 48 kHz signed PCM24 output, beat semantics, and reverse symmetry.

The browser estimate is `E + 3D + F + X + W + 2C + 32 MiB`, where `W = 68 + 6 * finalFrames` and `C = 393,216` bytes. The streaming Rust/WASM guard remains 256 MiB and models `D + F + X + 2C + 16 MiB`.
